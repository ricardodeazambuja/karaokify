# karaokify — agent notes

CLI that removes/attenuates vocals from audio/video (Demucs + ffmpeg).
Spec lives in PLAN.md; the "AI-agent contract" section there is normative.

## Environment bootstrap

- Use the `local` conda env: `condalocal` (shell alias) or
  `conda activate local`. It already has torch 2.10 + CUDA, demucs 4.1, ffmpeg 5.1.
- Never `pip install torch`/full deps here — install the package with
  `pip install -e . --no-deps`.
- GPU (CUDA) is available on this machine; `--device auto` picks it up.
  Demucs weights cache: `~/.cache/torch/hub/checkpoints/`.

## Test commands

```bash
pytest -m "not slow"   # fast unit tests (no separation)
pytest                 # + end-to-end (runs Demucs on a 5 s generated fixture)
```

Fixtures are generated by `tests/fixtures/make_fixture.py` — no binary blobs
in the repo.

## Layout

- `src/karaokify/cli.py` — argparse, gain resolution, exit codes (0/1/2/3)
- `src/karaokify/pipeline.py` — probe → separate → mix → remux orchestration
- `src/karaokify/separate.py` — demucs.api wrapper (torch imported lazily)
- `src/karaokify/mux.py` — ffmpeg/ffprobe subprocess helpers
- `src/karaokify/manifest.py` — --json document + JSONL manifest
- `src/karaokify/schema.json` — normative schema for --json output

Rules: stdout is reserved for `--json` data; all logs go to stderr.
Breaking the JSON schema or exit-code contract requires a major version bump.
